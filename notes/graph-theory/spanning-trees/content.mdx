A connected graph may contain more edges than it needs for connectivity. A spanning tree keeps every vertex reachable while deleting all cyclic redundancy.

## Subgraphs

> **Definition.** A graph $H$ is a subgraph of a graph $G$ if
>
> $$
> V(H)\subseteq V(G)
> \quad\text{and}\quad
> E(H)\subseteq E(G).
> $$

In the spanning-tree theorem, the subgraphs we build keep the full vertex set and remove only edges.

## Spanning Trees

> **Definition.** Let $G$ be a graph. A subgraph $T$ of $G$ is a spanning tree of $G$ if
>
> $$
> V(T)=V(G)
> $$
>
> and $T$ is a tree.

A spanning tree contains all vertices of $G$, but it usually does not contain all edges of $G$.

## The Main Theorem

> **Theorem.** Every connected graph has a spanning tree.

The proof removes edges from cycles. This is the correct graph-induction move because removing an edge keeps the same vertex set. If the smaller graph has a spanning tree, that tree still spans the original graph.

## Proof By Induction On Edges

> **Induction Claim.** For every $n\in\mathbb{N}$ and every connected graph $G$, if $|E(G)|=n$, then $G$ has a spanning tree.

We prove the claim by induction on $n$.

### Base Case

Let $G$ be connected and suppose

$$
|E(G)|=0.
$$

Since $G$ is connected and has no edges, it has at most one vertex. Therefore $G$ has no cycles.

So $G$ is connected and acyclic, which means $G$ is a tree. Since $G$ is a subgraph of itself and $V(G)=V(G)$, the graph $G$ is a spanning tree of itself.

### Inductive Hypothesis

> **Inductive Hypothesis.** Fix $k\in\mathbb{N}$ and assume that every connected graph with $k$ edges has a spanning tree.

### Inductive Step

Let $G$ be a connected graph with

$$
|E(G)|=k+1.
$$

There are two cases.

### Case 1: $G$ Has No Cycles

Then $G$ is connected and acyclic, so $G$ is a tree. Therefore $G$ is a spanning tree of itself.

### Case 2: $G$ Has a Cycle

Choose a cycle in $G$, and choose an edge $e$ on that cycle.

Define

$$
H\coloneqq G-e.
$$

Since $e$ lies on a cycle and $G$ is connected, the cycle-edge theorem gives that $H$ is connected.

Also,

$$
|E(H)|=k.
$$

By the inductive hypothesis, $H$ has a spanning tree. Call it $T$.

Since $H$ was formed from $G$ by deleting only an edge,

$$
V(H)=V(G).
$$

Since $T$ spans $H$,

$$
V(T)=V(H)=V(G).
$$

Also, $T$ is a tree and $E(T)\subseteq E(H)\subseteq E(G)$. Therefore $T$ is a spanning tree of $G$.

This completes the inductive step. Hence every connected graph has a spanning tree.

## Finding a Spanning Tree

To find a spanning tree of a connected graph:

1. If the graph has no cycles, stop. It is already a tree.
2. If the graph has a cycle, delete one edge from that cycle.
3. Repeat until no cycles remain.

The graph stays connected at every step because only cycle edges are deleted. Since the graph is finite, the process stops. The final graph is connected, acyclic, and has the original vertex set, so it is a spanning tree.

## Core Summary

A spanning tree is a tree subgraph with all the original vertices. Every connected graph has one: repeatedly delete an edge from a cycle. Cycle-edge deletion preserves connectedness, and deleting edges cannot remove vertices. When no cycle remains, the connected graph left behind is a tree.
