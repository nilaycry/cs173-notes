Problem Set 0 is the first place where the course begins testing more than arithmetic familiarity.

It is training precision about language, domain, and truth evaluation.

## What the Set Is Testing

The problems in Problem Set 0 press on three questions:

1. Is the expression under discussion the right kind of sentence?
2. If it is a sentence, what does it actually claim?
3. Does the sentence fit the ordinary true-or-false model cleanly?

That is why the set moves from routine mathematical claims to self-referential examples. The shift is intentional.

## Domain Precision in Ordinary Statements

Some early questions look elementary:

- every integer is either even or odd
- every number is either even or odd

Their purpose is not merely to review parity. They force attention to the domain of discourse.

"Every integer is either even or odd" may be true, while "Every number is either even or odd" is false, because the second statement ranges over a larger class of objects. A strong writeup should make that difference explicit rather than treating the two sentences as interchangeable.

## Self-Reference and Unstable Evaluation

The unusual sentences in Problem Set 0 are there to interrupt a false assumption:

grammatical sentence implies straightforward truth value.

That implication fails.

Some self-referential sentences can be assigned either truth value without contradiction. Some lead to contradiction under every attempted assignment. The important point is not philosophical terminology. The important point is that truth evaluation requires more care than the naive picture suggests.

## Expectations for Written Solutions

A complete solution to a typical Problem Set 0 item should do three things:

1. state the verdict
2. identify the relevant interpretation, domain, or structural fact
3. give the step that forces the verdict

For routine mathematical statements, that decisive step is often a counterexample or a short structural argument.

For self-referential statements, that decisive step is often the contradiction that follows from a proposed truth assignment.

## Common Errors

The most common errors in this set are:

- treating number and integer as interchangeable
- asserting that a sentence is false without giving the counterexample that makes it false
- calling a sentence paradoxical without showing the contradiction
- assuming every declarative sentence must have exactly one stable truth value

## Connection to the Rest of the Unit

The set prepares the later study of connectives and logical equivalence by forcing care about what is being evaluated in the first place. Once that care is in place, formal logical manipulation has a clear object.
